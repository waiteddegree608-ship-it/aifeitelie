# Friedrich QQ Bot (AI 腓特烈)

本项目是一个基于 QQ NT 客户端和桌面 UI 自动化的智能聊天机器人。支持通过大语言模型 (DeepSeek, GPT, Moonshot 等) 进行多模态的消息处理和自动回复，并集成了通过 Soundpad 将语音流直接注入 QQ 麦克风通道的功能。

## 核心功能

- **自动化通信处理**：通过 UI 自动化监听并响应 QQ NT 消息，支持将 TTS 语音通过系统路由（如虚拟声卡）发送至对话。
- **多模型接口支持**：兼容 OpenAI 标准 API 格式，支持纯文本模型及基于 Moonshot VL 的图像/表情识别。
- **动态发送控制**：内置基于字符长度计算的发送延迟模拟机制；支持长文本切片多段发送与撤回动作（Tickle）操作。
- **记忆管理结构**：
  - 基于短时缓存日志与 JSON 长时存储。
  - 内置定时例程，通过调用 LLM 归纳记忆片段，并执行基于权重的淘汰算法。
- **常规功能模块**：
  - 基于正则备用的交互式记账系统。
  - 定时或周期性任务提醒模块。
- **本地 Web 管理端**：基于 Flask 建立。支持系统运行状态监控、API 与参数热修及本地提示词（Prompt）树管理。

## 部署运行

### 1. 基础要求
- 操作系统：Windows
- 运行环境：Python 3.9 - 3.12 
- 语音依赖（特定场景）：[Soundpad](https://leppsoft.com/soundpad/en/) 与任意虚拟音频线缆软件应用（如 VB-Cable）。

### 2. 启动流程
1. 初始化项目并在 Windows 桌面侧挂载登入完毕的 PC QQ (NT 版) 客户端。
2. 运行根目录下的 `Run.bat` 完成基础环境依赖部署与 Python 程序启动。
3. 在系统自动分配并弹出的本地 Web 后管中补全环境参数及模型密钥。
4. 保存配置并通过面板指令启动主进程序列。

## 定制化说明

- **Prompt 构建**：可在 `prompts/` 目录新增对应的 markdown 配置文件，系统会在上下文栈区自动组装。
- **情绪图库配置**：存放在 `emojis` 相关子级目录下（按照 happy, sad, angry 划分）。程序回环根据情感解析匹配输出对应目录媒体文件。
- **预设业务指令**：例如 `/记账 [详情]`、`/查账` 等。
